# 调研文档质量检查清单

## 内容完整性

- [ ] **概述**：1-2 句话说明技术价值和适用场景
- [ ] **核心发现**：3-5 个 bullet points，提炼关键信息
- [ ] **技术细节**：实现原理、架构设计清晰
- [ ] **优缺点分析**：平衡客观，不偏颇
- [ ] **适用场景**：明确列出适合/不适合的场景
- [ ] **最佳实践**：生产环境建议
- [ ] **参考资料**：完整可追溯

## 格式规范

- [ ] **YAML Frontmatter**：包含所有必需字段
  - [ ] title
  - [ ] date (YYYY-MM-DD)
  - [ ] topic
  - [ ] tags
  - [ ] status
  - [ ] related

- [ ] **文件命名**：`YYYY-MM-DD-topic.md`
- [ ] **文件位置**：`research/{topic}/`
- [ ] **代码块**：使用正确的语言标识
- [ ] **表格**：格式正确，对齐清晰

## 代码质量

- [ ] **示例可运行**：至少验证核心代码片段
- [ ] **类型注解**：Python 代码包含 type hints
- [ ] **错误处理**：示例包含异常处理
- [ ] **注释清晰**：关键逻辑有注释说明

## 信息准确性

- [ ] **版本信息**：标注技术版本 (如 Python 3.11+)
- [ ] **时间敏感**：标注信息时效性 (如"截至 2026-02")
- [ ] **来源可信**：优先官方文档、高星项目
- [ ] **数据可验证**：性能数据有来源

## 可读性

- [ ] **结构清晰**：层级不超过 4 级
- [ ] **语言简洁**：无冗余表述
- [ ] **术语一致**：全文术语统一
- [ ] **图表辅助**：复杂概念有图示

## 决策支持

- [ ] **结论明确**：有清晰的推荐/不推荐
- [ ] **理由充分**：推荐有数据/事实支撑
- [ ] **约束考虑**：考虑实际技术/业务约束
- [ ] **风险提示**：标注潜在风险

## 快速评分

| 维度 | 评分 (1-5) | 备注 |
|------|------------|------|
| 完整性 | /5 | |
| 准确性 | /5 | |
| 可读性 | /5 | |
| 实用性 | /5 | |
| **总分** | **/20** | |

**评分标准**：
- 18-20 分：优秀，可直接发布
- 15-17 分：良好，小修后发布
- 12-14 分：合格，需要修改
- <12 分：不合格，需要重写

## 发布前检查

- [ ] 总分 ≥ 15 分
- [ ] 无严重错误 (准确性问题)
- [ ] 至少 1 人 Review
- [ ] 已更新 SUMMARY.md 索引
